PanSpark

an assembly language that is inspired by the syntax of the Sparc assembly language, but simplified to a level where it can be easily understood by a human

PancakeSpark is primarily written in JavaScript, designed to be run in games that require basic computers or microcontroller device simulation, with the support of I/O operations and easy interactions with the game world itself.

the most basic example could be a basic controller in Minecraft, that has 1 input and 3 outputs

with being able to operate both digital and analogue redstone signals.

Differences and syntax sugar

  • You can define a comment by wrapping it in //, for example // this is a comment //

  • Just like in languages like C, to allow the entire program to be stored in a single line string after programming it, you have to end your line by writing ;, for example mov 5, crabs;

Terminology

  • variable, like in common programming languages it represents a place in memory where a specific value can be stored, in the most shrimple terms, it is a register that has an assigned name and a value

  • registers, just like a variable, but it an existing space in memory, where you can control preset operations within a controller, like a robot arm, led’s, displays etc.

Basic Operations

  • set , allows the user to create and change a variable or access and change a registers and its value in memory like this: set <value>, <variable or register name>;, for example: set 5, number_one; would set a numeric value of 5 in memory with the name of number_one, unlike other assembly languages it would also allow, or it would let you set a value of a register like this set 1, led0;, you can also use a value inside a variable or a register to set the new value of a variable, for example set result, number2;

  • cmp , allows to compare two variables and execute a function if the result is true, which redirects the execution and after completing it, if it wasn’t returned or redirected, it continues on the function it started, for example cmp number1, number2, run_piston; will check if %number1 is equal to number2 and if they are equal, the run_piston function will be executed

  • add, allows to add a value to existing variables or registers, for example add 9, num10; will add 9 to the variable num10, you can also add variable values to an another variable or a register, for example add speed, &minecart_speed; would add the value of speed to the register &minecart_speed

  • ret this is a simple return function, stops execution unfinished
